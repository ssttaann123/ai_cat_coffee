# 编码约定

## 通用原则

- 代码简洁，避免过度工程化
- 优先复用现有模式，避免无谓重写
- 修改后尽量执行验证步骤
- 变量和函数命名清晰，见名知意

## JavaScript/Node.js

- 使用 ES Module 或 CommonJS 保持项目内一致
- 异步操作优先使用 async/await
- 错误处理不能吞掉异常，至少要有日志输出

## 文件组织

- 猫咪档案放 `cats/` 目录
- CLI 脚本放 `scripts/` 目录
- 开发技巧放 `skills/` 目录
- 新增目录需更新 README.md 的项目结构

## 注释规范

- 只在逻辑不自明的地方加注释
- 不写废话注释（如 `// 定义变量`）
- 复杂业务逻辑需要说明"为什么"而不是"做了什么"
